# Статус проекта

## Что это за проект

Проект — это Colab-based workflow для транскрибации длинных аудио- и видеофайлов в Google Docs.

Целевой сценарий:
- выбрать источник файла;
- при необходимости извлечь аудио из видео;
- выполнить транскрибацию через ElevenLabs как основной провайдер;
- использовать OpenAI как fallback в предусмотренных сценариях;
- сохранить результат в Google Docs;
- защититься от повторной траты кредитов через manifest и import existing / backfill.

## Что подтверждено

Подтверждено на уровне текущего handoff и репозитория:

- проект ориентирован на работу из Google Colab;
- основной STT-провайдер — `ElevenLabs / scribe_v2`;
- fallback-провайдер — `OpenAI / gpt-4o-transcribe`;
- для некоторых сценариев со спикерами рассматривается `OpenAI / gpt-4o-transcribe-diarize`;
- секреты должны храниться в Colab Secrets / `userdata`, а не в коде;
- результат должен сохраняться в Google Docs;
- в репозитории уже есть `README.md` и docs-контур.

## Что считается текущей кандидатной логикой

По handoff основная кандидатная логика включает:

- выбор провайдера;
- режимы языка `Русский / Автоопределение`;
- опцию разделения по спикерам;
- smart split для ограничений OpenAI по размеру файла;
- overlap-aware merge;
- silence-aware split;
- защиту от падения при отсутствии `OPENAI_API_KEY`, если OpenAI не выбран.

## Что важно не перепутать

На текущем этапе нужно различать:

1. **Архитектурный контекст проекта** — он уже достаточно хорошо зафиксирован.
2. **Фактическое состояние кода в GitHub** — его нужно проверять отдельно, а не предполагать.

По handoff каноническим ориентиром назывался файл:
- `elevenlabs_api_provider_openai_improved_nokeycrash.py`

Резервной rollback-страховкой назывался файл:
- `elevenlabs_api_scaled_progress.py`

Но сами эти файлы должны подтверждаться отдельно фактическим содержимым репозитория.

## Текущий приоритет

Ближайшая практическая цель проекта:

- довести репозиторий до состояния, пригодного для нормального review;
- зафиксировать документацию и структуру;
- затем привязать к ней реальный кодовый артефакт;
- после этого уже масштабировать работу через Codex / PR / review workflow.
