# Настройка Colab, Secrets и зависимости

## Базовая среда выполнения

Текущая рабочая среда проекта — Google Colab.

Это означает:
- код должен быть пригоден для запуска в Colab;
- секреты не должны захардкоживаться в коде;
- интеграция с Google Drive и Google Docs — часть практического сценария, а не внешний бонус.

## Принцип хранения секретов

Секреты должны храниться в Colab Secrets / `userdata`.

Минимально важные ключи:
- `ELEVENLABS_API_KEY`
- `OPENAI_API_KEY` — только для OpenAI fallback

Базовый шаблон доступа:

```python
from google.colab import userdata

ELEVENLABS_API_KEY = userdata.get("ELEVENLABS_API_KEY")
OPENAI_API_KEY = userdata.get("OPENAI_API_KEY")
```

## Практические правила по секретам

Нельзя:
- коммитить ключи в `.py` или `.ipynb`;
- хранить ключи в README или docs;
- оставлять вывод ячеек, где ключи могли быть засвечены;
- коммитить credential-файлы без явной необходимости.

Нужно:
- использовать `userdata` / Colab Secrets;
- перед заливкой ноутбука в GitHub очищать outputs;
- держать credential-файлы вне репозитория, если нет отдельной безопасной договорённости.

## Что нужно проверить перед first code push

Перед загрузкой кода в GitHub желательно проверить:

1. Нет ли в ноутбуке сохранённых outputs.
2. Нет ли в коде прямых строк с API keys.
3. Нет ли приватных Google-ссылок, которые не должны жить в публичной или shared-документации.
4. Нет ли служебных JSON / credential-файлов в рабочей папке.
5. Не попадают ли в commit медиа, временные результаты, manifests и экспортные артефакты.

## Почему Colab и GitHub должны жить вместе

Colab остаётся удобной средой запуска.

GitHub нужен для:
- истории изменений;
- нормального diff и review;
- PR и code review;
- работы через Codex и агентные сценарии.

Практически это означает такую схему:
- Colab = место запуска и интерактива;
- GitHub = source of truth для кода и документации.

## Рекомендуемый формат репозитория

Когда код будет окончательно привязан к репо, предпочтителен такой принцип:

- `.py` — основной читаемый артефакт для review;
- `.ipynb` — оболочка для Colab и удобного запуска;
- `README.md` — входная точка;
- `docs/` — подробная документация;
- `.gitignore` — защита от мусора, outputs и секретов.
